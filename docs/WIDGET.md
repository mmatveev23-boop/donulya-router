# UI Виджет — где смотреть

UI-часть роутера (то что видит клоузер в реальном времени) живёт в **другом репо**: `donulya-ai-auditor`. Виджет шапки amoCRM объединяет:

- Светофор дисциплины «🚦 Касания» (дожим существующих сделок)
- Роутер «🔥 Входящие лиды» (распределение новых лидов)

## Live URL

🔗 **https://mmatveev23-boop.github.io/donulya-ai-auditor/closer-top-bar.html**

GitHub Pages, статический HTML/JS (single-page mockup). В production будет интегрирован в amoCRM Widgets API.

## Что увидишь на странице

### Демо-панель сверху (только в мокапе)

```
🧪 ДЕМО РОУТЕРА: [🔥 Новый лид (push)] [+1 в очередь (pull)] [+3 в очередь] [↺ Сброс]
```

В проде её не будет — события приходят от бекенда через WebSocket / Server-Sent Events.

### Основная полоса (всегда видна) — ДОЖИМ

```
⏰ 22% сделок просрочено   8🔴/2🟠/3🟡/24🟢   ⚠ Поток −50%       🟢📞☕  🤖 6.7 ↓0.4  🏆 #3  📋 13  ▼
```

- `⏰ 22%` — мой % грехов (зона клоузера = оранжевая)
- `8🔴/2🟠/3🟡/24🟢` — раскладка моих сделок по 4 категориям
- `⚠ Поток −50%` — следствие моей зоны: получаю каждого второго лида
- `🟢📞☕` — мой текущий статус для роутера (свободен / на линии / перерыв)
- `🤖 6.7 ↓0.4` — общий AI-балл моих касаний за неделю + тренд vs прошлая неделя
- `🏆 #3` — моё место в команде по дисциплине
- `📋 13` — сделок к разбору сегодня
- `▼` — развернуть виджет (откроется детальный список сделок)

### Подполоса роутера (видна только при событии) — ПЕРВИЧНЫЕ ЛИДЫ

```
🤖 РОУТЕР · ВХОДЯЩИЕ ЛИДЫ   [🔥 Лебедев Н. 4:57]   [🔥 3 →]
```

Появляется когда:
- Робот назначил мне новый лид (красный pulsing pill с обратным таймером)
- Или есть отложенная очередь pull (жёлтая pill с кнопкой «Взять»)

Когда нет ни лида ни очереди — подполоса скрыта.

### Развёрнутый вид (клик на ▼)

5 колонок сделок по категориям:

1. Большой счётчик «то что проседает» (приоритет 🔴 > 🟠 > 🟡 > 🟢)
2. 🟢 Зелёные (`<24ч без касания`) + кнопка «Показать все N →»
3. 🟡 Жёлтые (`24-36ч`)
4. 🟠 Оранжевые (`36-48ч, скоро 🔴`)
5. 🔴 Красные (`>48ч`)

В каждой колонке кнопка `🤖 AI-разбор →` открывает модалку с AI-оценкой касаний по сделкам этой категории.

## Что попробовать

1. **Push-лид:** жми `🔥 Новый лид (push)` → появится красный pill в подполосе с таймером 5:00.
2. **Клик по 🔥 pill** = открывает карточку сделки в amoCRM (новая вкладка). **Таймер при этом НЕ сбрасывается** — это просто навигация. Защита от лайфхака.
3. **Сброс таймера = симуляция звонка Sipuni:** жми `📞 Sipuni: позвонил` → таймер сброшен, pill зеленеет «✓ принят», исчезает.
4. **Не нажимай ничего 5 минут** → таймер истечёт → pill станет серым «упустил» и исчезнет.
5. **Pull-очередь:** жми `+3 в очередь` → появится жёлтая pill «🔥 3 →» → клик = взять следующего (запускается обычный 5-мин таймер, требующий Sipuni-звонка для сброса).
6. **Status toggle:** переключи на 📞 «На линии» → попробуй симулировать новый лид → роутер скажет «ты не доступен».
7. **Зоны клоузера:** в самом верху страницы 4 кнопки (Лим 🟢 / Петрова 🟡 / Мифтахов 🟠 / Ляшенко 🔴) — переключай, увидишь как меняется полоса.

## Когда подключить к боевому бекенду

В production виджет получает реальные события от `donulya-router`:

| UI событие | Backend источник |
|---|---|
| Появление 🔥 pill | WebSocket push от роутера на `assignment.created` |
| Сброс таймера на 🔥 pill | Sipuni webhook `call_initiated` → роутер шлёт `assignment.engaged` |
| Появление queue-pull | `assignment.deferred` + worker в 9:00 пушит `queue.ready` |
| Обновление зоны клоузера | hourly worker пересчитывает + пишет в amoCRM custom_field + WebSocket push |

Технически в проде используется или **WebSocket** (постоянное соединение) или **Server-Sent Events** (HTTP стрим). В моке — `setInterval` и `simulateXxx()` функции для UX-теста.

## Связь с роутером

Виджет читает текущее состояние клоузера через endpoint:

```
GET /api/v1/widget/me
→ {
  user: { id, name, zone: 'orange' },
  active_assignment: { id, lead_name, expires_at },
  queued_count: 3,
  self_status: 'available',
  pipeline_state: { active: 37, green: 24, yellow: 3, orange: 2, red: 8 },
  ai_score_avg: 6.7,
  ai_score_trend: -0.4
}
```
